How It Works

Four-Layer
Detection Pipeline

Raw logs flow through a sequential architecture — from VM simulation to AI classification — ensuring no threat slips through undetected.

LAYER 01
🖥️
Data Collection & Preprocessing
Event logs generated from AMIDES & SOCBED across Ubuntu and Windows VMs, distributed and collected via Ansible automation. Python scripts clean, normalize, and encode log data.
AMIDES SOCBED Ansible Python
LAYER 02
🗄️
Storage & Labelling
Preprocessed logs stored in MongoDB (training) and Elasticsearch (testing). Static SIEM rulesets label each entry as benign or malicious before feeding the AI model.
MongoDB Elasticsearch SIEM Rules
LAYER 03
🧠
Deep Learning Model
A Deep Neural Network (DNN) with multiple hidden layers learns attack patterns from labeled data. Adam optimizer and Cross-Entropy loss drive accuracy across epochs.
DNN TensorFlow Keras ReLU / Softmax
LAYER 04
📡
Testing, Visualization & Alerting
Trained model classifies live logs from Elasticsearch. Results feed back into SIEM for correlation. A ReactJS + TailwindCSS GUI displays real-time threats with automated alerts.
ReactJS Kibana TailwindCSS

Layer 1
Virtualized Log Environments
Multi-OS VMs simulate realistic enterprise network conditions, generating both benign and attack event logs for diverse dataset creation.
Layer 2
Hybrid Storage Strategy
MongoDB stores labeled training datasets with flexible schema; Elasticsearch enables millisecond-speed querying of live log streams during inference.
Layer 3
Adaptive Neural Network
DNN with input, hidden, and output layers optimized with Adam gradient descent. Continuously retrained on new threat data to counter evolving attacks including zero-day exploits.
Layer 4
Visualization & Response
Kibana dashboards and a custom React GUI expose real-time threat maps, alert histories, and performance metrics. Automated alerts reduce MTTD/MTTR dramatically.

Threat Intelligence

Detected Threat Vectors

The DNN model is trained to identify a wide spectrum of attack patterns — from high-frequency brute-force attempts to subtle long-term data exfiltration campaigns.

CRITICAL
💥
Brute-Force Login
Rapid repeated authentication failures from a single IP. Detected via login frequency thresholds and temporal clustering in SIEM rules.
CRITICAL
🦠
Malware Infiltration
Suspicious process spawning and file system modifications flagged through behavioral anomaly detection in the DNN model.
HIGH
📤
Data Exfiltration
Unusual outbound data transfer volumes and access to restricted files from unauthorized hosts indicate active data theft.
HIGH
🎣
Phishing Activity
Fake URL and domain-based attack patterns identified through log correlation with known phishing indicators and network traffic analysis.
HIGH
🔍
Network Reconnaissance
Port enumeration and scanning activities detected through unusual connection patterns and high-frequency SYN packets across subnets.
MEDIUM
🔐
Privilege Escalation
Unauthorized attempts to gain elevated system access detected via user behavior anomaly modeling and access control log patterns.

Key Outcomes

Why AI
Changes Everything

Traditional SIEM systems rely on static rules — they miss zero-day attacks and generate excessive false alarms. The DNN learns non-linear, complex patterns that rule engines simply cannot express.

Adaptive Learning
The model continuously retrains on new log data from AMIDES/SOCBED and live SIEM outputs — maintaining high accuracy as attack patterns evolve over time.
Real-Time Response
From log ingestion to alert generation happens in seconds. Automated alerting drastically reduces Mean Time to Detect (MTTD) and Mean Time to Respond (MTTR).
